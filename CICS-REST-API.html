📋 Pré-requisitos

A SIT do CICS precisa ter TCPIP=YES para habilitar o suporte a HTTP. Verifique também se o DNS está configurado no arquivo DFHXXX.TCPIP.DATA com NSINTERADDR 8.8.8.8.

🟣 Parte 1: Servindo uma API REST

🎯 Objetivo: Criar um endpoint /api/v1/cics/info que retorna um JSON com informações do ambiente CICS (job, usuário, data, hora).

1.1 TCPIPSERVICE - O Listener HTTP

O TCPIPSERVICE é o recurso que escuta em uma porta TCP e recebe as requisições HTTP.

 CEDA  View TCpipservice( HTTP8080 )  
 TCpipservice   : HTTP8080           
 GROup          : GAPI               
 Urm            : DFHWBAAX           
 POrtnumber     : 08080              
 STatus         : Open               
 PROtocol       : Http               
 TRansaction    : CWXN               
 Backlog        : 00001              
 Host           : ANY                
 Ipaddress      : ANY                
 Maxdatalen     : 000032
🔑 Pontos Chave:
  • URM(DFHWBAAX) - Analyzer padrão que roteia para URIMAPs
  • TRANSACTION(CWXN) - Transação padrão do CICS Web Support
  • PORTNUMBER(08080) - Porta onde o CICS vai escutar

Comandos para criar:

CEDO DEFINE TCPIPSERVICE(HTTP8080) GROUP(GAPI) +
    URM(DFHWBAAX) +
    PORTNUMBER(08080) +
    STATUS(OPEN) +
    PROTOCOL(HTTP) +
    TRANSACTION(CWXN) +
    BACKLOG(00001) +
    HOST(ANY) +
    MAXDATALEN(000032)

CEDA INSTALL TCPIPSERVICE(HTTP8080) GROUP(GAPI)

1.2 URIMAP Server - O Roteador

A URIMAP com USAGE(SERVER) mapeia um PATH HTTP para um programa COBOL.

 CEDA  View Urimap( APISERVE )       
 Urimap         : APISERVE          
 Group          : GAPI              
 STatus         : Enabled           
 USAge          : Server            
 SCheme         : HTTP              
 HOST           : *                 
 PAth           : /api/v1/cics/info 
 TRansaction    : SAPI              
 PRogram        : APISERVE
🔑 Pontos Chave:
  • USAGE(SERVER) - Define que o CICS vai responder requisições
  • PATH(/api/v1/cics/info) - Rota que será exposta
  • TRANSACTION(SAPI) - Transação que executa o programa

Comandos para criar:

CEDA DEFINE URIMAP(APISERVE) GROUP(GAPI) +
    USAGE(SERVER) +
    SCHEME(HTTP) +
    HOST(*) +
    PATH(/api/v1/cics/info) +
    TRANSACTION(SAPI) +
    PROGRAM(APISERVE)

CEDA INSTALL URIMAP(APISERVE) GROUP(GAPI)

1.3 Transação

A transação SAPI é acionada quando a URIMAP recebe uma requisição.

 CEDA  View TRANSaction( SAPI )     
 TRANSaction    : SAPI             
 Group          : GAPI             
 PROGram        : APISERVE

Comandos para criar:

CEDA DEFINE TRANSACTION(SAPI) GROUP(GAPI) +
    PROGRAM(APISERVE)

CEDA INSTALL TRANSACTION(SAPI) GROUP(GAPI)

1.4 Programa COBOL - APISERVE

O programa que processa a requisição e retorna o JSON.

       IDENTIFICATION DIVISION.
       PROGRAM-ID. APISERVE.

       DATA DIVISION.
       WORKING-STORAGE SECTION.

       01 WS-RESPONSE-CODE     PIC S9(8) COMP.
       01 WS-RESPONSE-CODE2    PIC S9(8) COMP.

      * Dados recebidos
       01 WS-REQUEST-DATA      PIC X(500).
       01 WS-REQUEST-LEN       PIC S9(8) COMP.

      * Dados de resposta
       01 WS-RESPONSE-JSON     PIC X(500).
       01 WS-RESPONSE-LEN      PIC S9(8) COMP.

      * Headers HTTP
       01 WS-MEDIA-TYPE        PIC X(50) 
           VALUE 'application/json'.
       01 WS-CHARSET           PIC X(12) VALUE 'UTF-8'.
       01 WS-HTTP-STATUS       PIC S9(8) COMP VALUE 200.
       01 WS-HTTP-STATUS-MSG   PIC X(20) VALUE 'OK'.

      * Dados do CICS
       01 WS-CICS-JOB          PIC X(8).
       01 WS-CICS-USER         PIC X(8).
       01 WS-CICS-TIME         PIC X(8).
       01 WS-CICS-DATE         PIC X(8).

      * Variáveis auxiliares
       01 WS-TEMP-LEN          PIC S9(8) COMP.

       PROCEDURE DIVISION.

      * Obter informações do ambiente CICS
           EXEC CICS ASSIGN
               USERID(WS-CICS-USER)
               APPLID(WS-CICS-JOB)
               RESP(WS-RESPONSE-CODE)
           END-EXEC

           IF WS-RESPONSE-CODE NOT = DFHRESP(NORMAL)
               MOVE 'CICSA02' TO WS-CICS-JOB
               MOVE 'CICSUSR' TO WS-CICS-USER
           END-IF

           MOVE FUNCTION CURRENT-DATE(1:8) TO WS-CICS-DATE
           MOVE FUNCTION CURRENT-DATE(9:6) TO WS-CICS-TIME

      * Receber dados da requisição (se houver)
           EXEC CICS WEB RECEIVE
               INTO(WS-REQUEST-DATA)
               MAXLENGTH(500)
               LENGTH(WS-REQUEST-LEN)
               RESP(WS-RESPONSE-CODE)
           END-EXEC

      * Montar JSON de resposta
           MOVE SPACES TO WS-RESPONSE-JSON

           STRING 
               '{"status":"ok","message":"API CICS OK!",'
               '"data":"'          WS-CICS-DATE '"'
               ',"time":"'         WS-CICS-TIME(1:6) '"'
               ',"cics_job":"'     WS-CICS-JOB  '"'
               ',"cics_user":"'    WS-CICS-USER '"'
               ',"timestamp":"2026-06-02T00:00:00Z"}'
               DELIMITED BY SIZE 
               INTO WS-RESPONSE-JSON
           END-STRING

      * Calcular tamanho real
           MOVE 0 TO WS-RESPONSE-LEN
           MOVE 500 TO WS-TEMP-LEN
           PERFORM UNTIL WS-TEMP-LEN = 0
               IF WS-RESPONSE-JSON(WS-TEMP-LEN:1) NOT = SPACE
                   MOVE WS-TEMP-LEN TO WS-RESPONSE-LEN
                   MOVE 0 TO WS-TEMP-LEN
               ELSE
                   SUBTRACT 1 FROM WS-TEMP-LEN
               END-IF
           END-PERFORM

           IF WS-RESPONSE-LEN = 0
               MOVE 160 TO WS-RESPONSE-LEN
           END-IF

      * Enviar resposta HTTP
           EXEC CICS WEB SEND
               FROM(WS-RESPONSE-JSON)
               FROMLENGTH(WS-RESPONSE-LEN)
               MEDIATYPE(WS-MEDIA-TYPE)
               CHARACTERSET(WS-CHARSET)
               STATUSCODE(WS-HTTP-STATUS)
               STATUSTEXT(WS-HTTP-STATUS-MSG)
               STATUSLEN(2)
               RESP(WS-RESPONSE-CODE)
               RESP2(WS-RESPONSE-CODE2)
           END-EXEC

           DISPLAY 'Resposta enviada: ' WS-RESPONSE-JSON(1:WS-RESPONSE-LEN)

           EXEC CICS RETURN END-EXEC.
🎯 Segredo da Conversão EBCDIC→UTF-8: O parâmetro CHARACTERSET(WS-CHARSET) com valor 'UTF-8' instrui o CICS a converter automaticamente os dados de EBCDIC para UTF-8. Sem isso, o cliente receberia caracteres ilegíveis!

🔵 Parte 2: Consumindo uma API REST

🎯 Objetivo: Consumir a API pública api.open-notify.org/iss-now.json que retorna a posição atual da Estação Espacial Internacional (ISS).

2.1 URIMAP Client - Conexão com API Externa

A URIMAP com USAGE(CLIENT) define como o CICS se conecta a uma API externa.

 CEDA  View Urimap( URIGET   )        
 Urimap         : URIGET             
 Group          : GAPI               
 STatus         : Enabled            
 USAge          : Client             
 SCheme         : HTTP               
 POrt           : 00080              
 HOST           : api.open-notify.org
 PAth           : /iss-now.json      
                :
🔑 Pontos Chave:
  • USAGE(CLIENT) - Define que o CICS vai consumir a API
  • SCHEME(HTTP) - Usa HTTP (sem TLS/SSL)
  • HOST(api.open-notify.org) - Nome do host da API
  • PORT(00080) - Porta 80 (HTTP padrão)

Comandos para criar:

CEDA DEFINE URIMAP(URIGET) GROUP(GAPI) +
    USAGE(CLIENT) +
    SCHEME(HTTP) +
    HOST(api.open-notify.org) +
    PORT(00080) +
    PATH(/iss-now.json)

CEDA INSTALL URIMAP(URIGET) GROUP(GAPI)

2.2 Transação

A transação ISSG executa o programa que faz a chamada à API externa.

 CEDA  View TRANSaction( ISSG )       
 TRANSaction    : ISSG               
 Group          : GAPI               
 PROGram        : TESTGET

Comandos para criar:

CEDA DEFINE TRANSACTION(ISSG) GROUP(GAPI) +
    PROGRAM(TESTGET)

CEDA INSTALL TRANSACTION(ISSG) GROUP(GAPI)

2.3 Programa COBOL - TESTGET

O programa que consome a API externa via GET e exibe a resposta.

       IDENTIFICATION DIVISION.
       PROGRAM-ID. TESTGET.

       DATA DIVISION.
       WORKING-STORAGE SECTION.

       01 WS-RESPONSE-CODE     PIC S9(8) COMP.
       01 WS-RESPONSE-CODE2    PIC S9(8) COMP.
       01 WS-TOKEN             PIC X(8).

      * Dados da resposta
       01 WS-JSON-RESPONSE     PIC X(2000).
       01 WS-RESP-LEN          PIC S9(8) COMP.

      * HTTP Status
       01 WS-HTTP-STATUS       PIC S9(8) COMP.
       01 WS-HTTP-STATTEXT     PIC X(100).
       01 WS-STATTEXT-LEN      PIC S9(8) COMP VALUE 100.

       PROCEDURE DIVISION.

           DISPLAY 'Consultando ISS Location...'

      * 1. Abrir conexao HTTP
           EXEC CICS WEB OPEN
               URIMAP('URIGET')
               SESSTOKEN(WS-TOKEN)
               RESP(WS-RESPONSE-CODE)
               RESP2(WS-RESPONSE-CODE2)
           END-EXEC

           IF WS-RESPONSE-CODE NOT = DFHRESP(NORMAL)
               DISPLAY 'Erro WEB OPEN'
               DISPLAY 'RESP : ' WS-RESPONSE-CODE
               DISPLAY 'RESP2: ' WS-RESPONSE-CODE2
               EXEC CICS RETURN END-EXEC
           END-IF

           DISPLAY 'WEB OPEN OK - Token: ' WS-TOKEN

      * 2. Enviar GET e receber resposta
           EXEC CICS WEB CONVERSE
               SESSTOKEN(WS-TOKEN)
               METHOD(DFHVALUE(GET))
               INTO(WS-JSON-RESPONSE)
               MAXLENGTH(2000)
               TOLENGTH(WS-RESP-LEN)
               STATUSCODE(WS-HTTP-STATUS)
               STATUSTEXT(WS-HTTP-STATTEXT)
               STATUSLEN(WS-STATTEXT-LEN)
               RESP(WS-RESPONSE-CODE)
               RESP2(WS-RESPONSE-CODE2)
           END-EXEC

           DISPLAY '============================================'
           DISPLAY 'RESULTADO:'
           DISPLAY 'CICS RESP : ' WS-RESPONSE-CODE
           DISPLAY 'CICS RESP2: ' WS-RESPONSE-CODE2
           DISPLAY 'HTTP STATUS: ' WS-HTTP-STATUS
           DISPLAY 'RESP DADOS : ' 
               WS-JSON-RESPONSE(1:WS-RESP-LEN)
           DISPLAY '============================================'

      * 3. Fechar conexao
           EXEC CICS WEB CLOSE
               SESSTOKEN(WS-TOKEN)
               RESP(WS-RESPONSE-CODE2)
           END-EXEC

           DISPLAY 'Programa finalizado.'
           EXEC CICS RETURN END-EXEC.

Práticas de Mercado

O CICS TS possui suporte nativo a HTTP, permitindo criar e consumir APIs REST com COBOL. Para APIs HTTPS, é necessário um proxy como Flask, nginx, IBM Integration Bus (IIB) ou IBM z/OS Connect que faça a ponte entre o HTTP do CICS e o HTTPS da internet. Esta arquitetura é o padrão de modernização de mainframes no mercado.


Abaixo uma tabela comparativa das ferramentas.

Solução Tipo Complexidade Custo Ideal para
IBM z/OS Connect Específico para CICS Média $$$ (IBM) Mainframe puro
IBM Integration Bus (IIB) ESB completo Alta $$$$ (IBM) Integração corporativa

nginx / HAProxy Proxy reverso Baixa Grátis Produção
