🚀 Integrando Mainframe e Plataforma x86 com IBM MQ

Autor: Diego Eduardo Moysés

Exemplo prático de mensageria entre ambientes Mainframe e x86 utilizando o IBM MQ.

📊 Arquitetura da Integração com MQ
💻 Mainframe (CSQ7)
Receiver: QM1.TO.CSQ7
Sender: CSQ7.TO.QM1
XMITQ: QM1.XMIT.QUEUE
Fila Requisição: LAB.LOCAL.APP.REQ.CICS (Trigger)
Fila Resposta: LAB.REMOTE.QM01.RECEBE.ZOS (QREMOTE)
🖥️ *Windows X86 (QM1)
Sender: QM1.TO.CSQ7
Receiver: CSQ7.TO.QM1
XMITQ: CSQ7.XMIT.QUEUE
Fila Envio: LAB.REMOTE.APP.REQ.CICS (QREMOTE)
Fila Resposta: LAB.APP.RECEBE.ZOS (QLOCAL)

1. No ambiente Mainframe vamos definir.

💡 Nota: Execute os comandos abaixo no console do MQ Series (CSQ7) no Mainframe.

Canal Sender:

DEFINE CHANNEL('CSQ7.TO.QM1') +
       CHLTYPE(SDR) +
       TRPTYPE(TCP) +
       CONNAME('10.0.0.2(1415)') +
       XMITQ('QM1.XMIT.QUEUE') +
       REPLACE

Canal Receiver:

DEFINE CHANNEL('QM1.TO.CSQ7') +
       CHLTYPE(RCVR) +
       TRPTYPE(TCP) +
       REPLACE

Fila XMIT:

DEFINE QLOCAL('QM1.XMIT.QUEUE') +
       USAGE(XMITQ) +
       REPLACE

Fila de requisições:

Observação: esta fila irá acionar via trigger a transação CICS que irá processar a mensagem e postar a resposta. Mais detalhes de como configurar fila trigger com CICS pode ser encontrado em Mensageria com Job Batch, Transação CICS e Trigger.

DEFINE QLOCAL('LAB.LOCAL.APP.REQ.CICS') +
       USAGE(NORMAL) +
       TRIGGER +
       TRIGTYPE(FIRST) +
       TRIGDPTH(1) +
       INITQ('CSQ7.TRIGGER.CICSA02') +
       PROCESS('LAB.PROCESS.CICS.WIN') +
       PUT(ENABLED) +
       GET(ENABLED) +
       SHARE +
       DEFSOPT(SHARED) +
       REPLACE

Fila de resposta:

DEFINE QREMOTE('LAB.REMOTE.QM01.RECEBE.ZOS') +
       RNAME('LAB.APP.RECEBE.ZOS') +
       RQMNAME('QM1') +
       XMITQ('QM1.XMIT.QUEUE') +
       PUT(ENABLED) +
       REPLACE

2. Ambiente x86 Windows

💡 Nota: Execute estes comandos no runmqsc QM1 do Windows.

Canal Sender:

DEFINE CHANNEL('QM1.TO.CSQ7') +
       CHLTYPE(SDR) +
       TRPTYPE(TCP) +
       CONNAME('10.0.0.1(1414)') + 
       XMITQ('CSQ7.XMIT.QUEUE') +
       DESCR('Canal para o ZOS') +
       REPLACE

Canal Receiver:

DEFINE CHANNEL('CSQ7.TO.QM1') +
       CHLTYPE(RCVR) +
       TRPTYPE(TCP) +
       SSLCAUTH(OPTIONAL) +
       DESCR('RECEIVER DO ZOS') +
       REPLACE

Fila XMIT:

DEFINE QLOCAL('CSQ7.XMIT.QUEUE') +
       USAGE(XMITQ) +
       DESCR('Fila XMIT para o Mainframe') +
       REPLACE

Fila para enviar mensagem ao Mainframe:

DEFINE QREMOTE('LAB.REMOTE.APP.REQ.CICS') +
       RNAME('LAB.LOCAL.APP.REQ.CICS') +
       RQMNAME('CSQ7') +
       XMITQ('CSQ7.XMIT.QUEUE') +
       PUT(ENABLED) +
       DESCR('Fila remota para o ZOS CSQ7') +
       REPLACE

Fila para receber a reposta:

DEFINE QLOCAL('LAB.APP.RECEBE.ZOS') +
       USAGE(NORMAL) +
       DESCR('Recebe mensagens do ZOS') +
       PUT(ENABLED) +
       GET(ENABLED) +
       SHARE +
       REPLACE

3. Programa COBOL CICS

O programa MQRESWIN irá ler a mensagem postada pelo MQ Windows e devolver uma mensagem de resposta. O Fonte completo pode ser acessado aqui. 

       IDENTIFICATION DIVISION.
       PROGRAM-ID. MQRESWIN.

       DATA DIVISION.
       WORKING-STORAGE SECTION.

       01  MQ-OD-AREA.
           COPY CMQODV.
       01  MQ-MD-AREA.
           COPY CMQMDV.
       01  MQ-GMO-AREA.
           COPY CMQGMOV.
       01  MQ-CONST-AREA.
           COPY CMQV.
       01  MQ-PMO-AREA.
           COPY CMQPMOV.

       01  WS-QUEUE-NAME-TEXT  PIC X(48) VALUE SPACES.
       01  WS-MSG-LEN          PIC S9(9) COMP VALUE 56.
       01  WS-MSG-DATA-LEN     PIC S9(9) COMP VALUE 0.
       01  WS-HCONN            PIC S9(9) COMP VALUE 0.
       01  WS-HOBJ             PIC S9(9) COMP VALUE 0.
       01  WS-COMP-CODE        PIC S9(9) COMP VALUE 0.
       01  WS-REASON-CODE      PIC S9(9) COMP VALUE 0.
🔑 Ponto Chave: Este programa usa as copylibs padrão do MQ (CMQODV, CMQMDV, CMQGMOV, CMQV, CMQPMOV) e implementa o fluxo MQGET → Processamento → MQPUT para responder ao Windows.

4. Transação no CICS

Definir e instalar a transação WINX apontando para o programa MQRESWIN

CEDA DEFINE TRANSACTION('WINX') -
       GROUP('GAOR') -
       PROGRAM('MQRESWIN') -
       STATUS(ENABLED) -
       ISOLATE(YES) -
       REPLACE
⚠️ Atenção: caso não esteja em ambiente configurado para autoinstall (PROGAUTOINST) de programas na chamada, defina também o programa no CICS: 
CEDA DEFINE PROGRAM('MQRESWIN') -
       GROUP('GAOR') -
       LANGUAGE(COBOL) -
       STATUS(ENABLED) -
       REPLACE
